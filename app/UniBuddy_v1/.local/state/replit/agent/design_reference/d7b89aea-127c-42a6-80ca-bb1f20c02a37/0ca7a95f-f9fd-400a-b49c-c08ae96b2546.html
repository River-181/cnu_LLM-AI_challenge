📋 강의 개요

이번 강의에서는 관계형 데이터베이스의 핵심 개념인 정규화(Normalization)에 대해 학습합니다. 데이터 중복을 제거하고 무결성을 보장하는 방법을 다룹니다.

학습 목표

  • 정규화의 개념과 필요성 이해
  • 1NF, 2NF, 3NF의 특징 학습
  • 함수적 종속성 파악 능력

핵심 키워드

정규화 함수적 종속 이상현상 기본키 부분종속

1 정규화의 개념

정규화는 관계형 데이터베이스에서 데이터의 중복을 최소화하고 데이터 무결성을 확보하기 위해 데이터를 구조화하는 과정입니다.

이상현상 (Anomaly)

  • • 삽입 이상: 불필요한 데이터 입력
  • • 수정 이상: 데이터 불일치 발생
  • • 삭제 이상: 의도하지 않은 정보 손실

2 함수적 종속성

한 속성의 값이 다른 속성의 값을 유일하게 결정하는 관계를 의미합니다. A → B로 표현하며, A가 B를 함수적으로 종속시킨다고 말합니다.

예시

학생ID → 학생이름, 전공
(학생ID가 주어지면 이름과 전공이 유일하게 결정됨)

3 제1정규형 (1NF)

테이블의 모든 속성이 원자값(Atomic Value)을 가져야 합니다. 즉, 하나의 셀에는 하나의 값만 저장되어야 합니다.

❌ 위반 예시

취미: "축구, 농구, 야구"

✅ 올바른 예시

취미: "축구"
취미: "농구"

4 제2정규형 (2NF)

1NF를 만족하고, 모든 비주요 속성이 기본키에 완전 함수적 종속되어야 합니다. 부분 종속성을 제거해야 합니다.

해결 방법

부분 종속 속성들을 별도 테이블로 분리하여 완전 함수적 종속 관계를 만듭니다.
